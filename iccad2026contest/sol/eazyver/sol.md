# eazyver 方案说明

## 目标

本版本只考虑：

- 所有硬约束必须满足
  - 无重叠
  - 普通软块面积满足 `w * h = area_target`
  - `fixed-shape` 的 `w, h` 不变
  - `preplaced` 的 `x, y, w, h` 不变
- 优化目标只保留
  - 最小化最终包围盒面积

不考虑：

- `HPWL`
- `grouping / MIB / boundary`

## 核心拆分

把所有块分成两类：

1. 刚性块
   - `preplaced`
   - `fixed-shape`
2. 柔性块
   - 其余普通软块

关键观察：

- 柔性块长宽比无限制，因此只要给它一个矩形空地，且空地面积不小于它的目标面积，就一定能构造出合法 `(w, h)`。
- 真正决定包围盒面积下限的，是刚性块的相对排布。

因此实现采用：

`先压缩刚性骨架，再把软块填进骨架空洞，最后把剩余软块挂到外侧条带。`

## 刚性骨架求解

### 1. 固定部分

- 所有 `preplaced` 直接按输入坐标锁死
- 所有 `fixed-shape` 只允许改 `x, y`

### 2. 可移动刚性块的布局

对所有可移动刚性块做一个小规模离散搜索：

- 先按面积/边长做若干种排序
- 对每一种排序做 beam search
- 每次放一个块时，只枚举和当前已放块边界对齐的候选位置
  - 左边对齐已有块左/右边
  - 右边对齐已有块左/右边
  - 下边对齐已有块上/下边
  - 上边对齐已有块上/下边
- 候选位置只保留无重叠的
- 用当前刚性骨架的 bbox 面积、周长、最长边作为排序指标

beam search 完成后，再做几轮 remove-and-reinsert 局部压缩。

这样做的原因是：

- 验证集里刚性块数量不大
- 这个简化版本里软块不需要进主搜索
- 只搜索刚性骨架，性价比最高

## 软块填充

刚性骨架确定后：

1. 先求刚性骨架 bbox
2. 用刚性块边界把 bbox 切成网格
3. 找出 bbox 内所有空矩形
4. 按面积从大到小，把软块优先塞进这些空矩形
   - 每次都把软块切成“整宽横条”或“整高竖条”
   - 这样剩余空间仍然是矩形，后续继续可递推
5. 如果还有剩余软块，就把它们挂到 bbox 外侧
   - 若骨架更宽，则挂到顶部
   - 若骨架更高，则挂到右侧

这样保证：

- 所有块都可构造
- 无重叠
- 所有面积都严格满足

## 退化情形

如果没有刚性块，说明全是普通软块。

此时直接把所有块排成同高条带：

- 取公共高度 `H`
- 第 `i` 个块设为 `h_i = H, w_i = a_i / H`

那么最终包围盒面积恰好等于总面积 `sum(a_i)`，这是该情形下的最优值。

## 实现要点

- 不旋转 `fixed-shape`
- `preplaced` 不平移
- 所有坐标都允许浮点
- 软块面积按公式直接构造，尽量避免误差积累
- 只用 bbox area 选解，不混入 HPWL 和软约束项

## 预期效果

这个版本的强项是：

- 简洁
- 对本简化目标贴合
- 刚性块少时通常能把面积压得比较紧

它的弱点也明确：

- 不关心任何软约束
- 不关心线长
- 刚性骨架搜索仍然是启发式，不是全局最优证明

但对当前要求，这个取舍是合理的。
