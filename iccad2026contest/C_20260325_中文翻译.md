## 背景

固定外框布局规划是一类 NP 完全的组合优化问题，已有大量文献研究。在工业级片上系统（SoC）设计中，由于模块形状和位置存在硬性约束，这一问题会复杂得多。目标是在二维画布上放置各个模块（也称为 block 或 module），使多目标代价函数最小，同时满足所有约束。

SoC 布局规划与传统装箱问题在若干关键方面不同。模块可能具有可变形状，但必须遵守预先给定的面积预算。某些模块还受特定放置约束限制。模块之间通过网络相连，并且往往需要连接到画布边界上的外部端点，这会形成空间依赖关系，直接影响总线长，而总线长是主要优化目标之一。该代价函数本质上是多目标的，需要同时最小化芯片包围盒面积和总线长，并满足所有放置约束。

## 动机

在物理设计后端流程中，面对多维约束空间时，如何缩短高复杂度设计的收敛时间已经成为关键瓶颈。本竞赛的目标是识别能够处理这些工业规模挑战的高效方法。

核心目标是将设计范式从当前通常需要数天的人工迭代，转变为几分钟内完成的自动化循环。我们假设，通过机器学习（ML）引导的优化最有可能实现这一转变。然而，此类模型的有效性高度依赖高保真、有标签的数据集。为弥补这一空缺，我们提供了一套源自真实工业场景的综合基准数据集。通过发布这些数据集（例如 FloorSet），我们希望推动社区开发能够扩展到工业需求的 ML 驱动解决方案。

我们的基本假设是，ML 引导能够快速探索解空间，并成为智能体式 AI 和对话式布局规划智能体的基础组件。从长期看，我们设想一种设计环境：架构师可以通过自然语言与布局规划智能体交互，并以低于一分钟的延迟完成设计迭代。

## 相关工作

带有多样放置约束的固定外框布局规划，是物理设计领域中成熟但日益复杂的挑战。过去，社区通常依赖 MCNC 和 GSRC 等传统基准来验证各种求解器，从经典元启发式算法到近年的混合学习框架均包括在内。然而，这些数据集缺乏现代机器学习验证所需的严谨性。

具体而言，它们不能提供真实最优解，而真实最优解在更广泛的机器学习领域中通常是标准要求；同时，它们往往孤立地处理约束，而不是把约束整合到一个完整设计问题中。

相比之下，本文提出的 FloorSet 数据集弥补了这一空缺。它提供了数百万个布局解，其中面积和线长可证明达到最优，并且所有放置约束都严格满足。通过采用一种反向工程方法，数据集在构造时即保证近似最优。这些基准建立了明确的真实最优参考，使任何候选求解器的最优性差距都可以被精确量化。

考虑到带软模块和直角多边形划分的完整工业复杂度会给神经网络架构带来较高门槛，我们引入 FloorSet-Lite。这个精选子集只关注硬模块，为评估下一代机器学习驱动的 EDA 工具提供了简化但高保真的环境。

## 概念验证

我们的初步实验表明，即便使用大规模并行的传统启发式方法，例如分布式模拟退火，在 FloorSet-Lite 数据集上也表现吃力。即使对于中等规模测试用例（例如 60 个划分），运行时间也常常超过 10 分钟，并且无法达到最优，在线长或面积方面至少存在 10% 差距。

我们内部使用扩散模型的实验给出了相反证据：它能够在一分钟以内获得高保真解。虽然所提供的数据集面向 ML 训练，但本竞赛的目标是性能导向，而不是限制方法类别。我们欢迎任何算法范式，包括纯随机方法、数据驱动方法或混合方法，只要能够有效处理解质量和运行时间之间的权衡。

## 输入

- `B = {b1, b2, ..., bk}` 表示 `k` 个模块的集合。每个模块 `bi` 是一个软模块，并受预定面积目标 `A = {ai | i = 1, 2, ..., k}` 约束。
- `T = {t1, t2, ..., tr}` 表示 `r` 个端点，它们是二维平面上预定义的固定点，用于外部接口。端点位置由输入给出，并在整个竞赛过程中保持固定。
- 模块间连接：加权邻接矩阵 `W(int) ∈ R^(k×k)`，其中 `W(int)ij` 是连接 `bi` 和 `bj` 的网络权重。权重为 0 表示无连接。
- 外部连接：加权邻接矩阵 `W(ext) ∈ R^(k×r)`，其中 `W(ext)ij` 是连接 `bi` 和端点 `tj` 的网络权重。权重为 0 表示无连接。
- 软约束：在本竞赛中，违反软约束会产生惩罚，但不会使解失效或不可行。

软约束包括：

- 分组约束：`Bgrouping^P` 定义 `P` 组模块，这些模块应在物理上相邻接，即共享一段长度非零的公共边且间隙为零。若一个组内所有模块通过共享边形成单个连通分量，则该组满足约束。
- 多实例模块（MIB）：`Bmib^Q` 定义 `Q` 组模块，这些模块应具有相同尺寸（宽度和高度）。它们代表同一 master cell 的不同实例，必须具有一致形状。
- 边界约束：`Bboundary` 指定若干模块，它们应至少有一条边接触包围盒边界（边约束），或有两条边接触包围盒角点（角约束）。输入会为每个模块给出具体要求的边或角。

硬约束包括：

- 面积目标和维度约束：所有模块尺寸必须严格遵守指定要求。对于预放置模块和固定形状模块，输入尺寸 `wi` 和 `hi` 不可改变。对于其他软模块，实现宽度 `wi` 和高度 `hi` 必须在 1% 相对误差阈值内满足目标面积 `ai`：

```text
|(wi * hi - ai) / ai| <= 0.01
```

任何解若偏离固定尺寸，或软模块面积误差超过 1%，都被判定为不可行。

- 无重叠约束：解必须严格无重叠。对于任意两个不同模块 `bi` 和 `bj`（`i != j`），它们交集面积必须为 0：

```text
Area(bi ∩ bj) = 0
```

模块几何体之间任何程度的相交都会使解不可行。注意：模块可以共享一条边，即相互接触，但不能重叠。

- 固定形状不可变约束：对于每个 `b ∈ Bfixed`，解中的尺寸必须与输入规格完全一致。目标面积 `ab` 被忽略，只看指定的宽度和高度。若任意固定形状模块不满足

```text
wb = winput_b 且 hb = hinput_b
```

则该解不可行。

注：该项在旧版文档中属于软约束；`v10` 已在文档和评测代码中同时改为硬约束。

- 预放置不可变约束：对于每个 `b ∈ Bpreplaced`，位置和尺寸都必须与输入规格完全一致。目标面积 `ab` 被忽略，只看指定的位置、宽度和高度。若任意预放置模块不满足

```text
(xb, yb) = (xinput_b, yinput_b)
且 (wb, hb) = (winput_b, hinput_b)
```

则该解不可行。

注：该项在旧版文档中属于软约束；`v10` 已在文档和评测代码中同时改为硬约束。

对于某个测试用例，不可行解会获得固定惩罚代价 `M = 10`，定义见目标函数。这保证任何可行解无论质量如何，得分都严格优于不可行解。

## 期望输出

- 无重叠模块位置：

```text
L = {(xi, yi) | i = 1, 2, ..., k}
```

其中 `(xi, yi)` 是模块 `bi` 左下角坐标。坐标系原点 `(0, 0)` 位于画布左下角，`x` 向右增加，`y` 向上增加。对于预放置模块，坐标 `(xi, yi)` 不可改变，并且必须与输入规范完全一致。

- 模块尺寸：

```text
D = {(wi, hi) | i = 1, 2, ..., k}
```

其中 `wi` 是模块 `bi` 的宽度（x 方向跨度），`hi` 是高度（y 方向跨度）。因此模块 `bi` 占据矩形区域：

```text
[xi, xi + wi] × [yi, yi + hi]
```

- 输出格式：解必须按照 FloorSet 仓库中指定的格式提交。文件格式细节和提交说明请参见仓库文档。

## 目标函数

本竞赛采用如下多目标代价函数：

```text
若可行：
Cost = min(
         (1 + α * (HPWLgap + Areagap_bbox))
         * exp(β * Violationsrelative)
         * max(0.7, RuntimeFactor^γ),
         M - 10^-6
       )

若不可行：
Cost = M
```

其中：

- 若解违反任何硬约束（模块重叠、面积容差违规、固定形状尺寸不匹配、或预放置模块的位置/尺寸不匹配），则不可行，并获得固定惩罚 `M = 10`。
- 对于可行解，其代价会被截断到 `M - 10^-6`，从而保证任意可行解的得分都严格优于不可行解。
- `α = 0.5`，用于加权质量指标（HPWL 和包围盒面积差距）。
- `β = 2.0`，控制软约束违规的指数惩罚。
- `γ = 0.3`，用于削弱运行时间因子的影响。
- `max(0.7, ·)` 将运行时间带来的收益上限限制为 30%。
- `HPWLgap` 是已实现线长与基线（最优）线长之间的相对差距。
- `Areagap_bbox` 是已实现包围盒面积与基线（最优）面积之间的相对差距。
- `Violationsrelative ∈ [0, 1]` 量化软约束违规程度。
- `RuntimeFactor = 你的运行时间 / 所有提交的中位运行时间`。该指标针对每个测试设计独立计算，以该测试用例自身的中位运行时间作为唯一参考。

解释：

- 可行解的代价通常在 `[0.7, 7.4]` 范围内。
- 最佳情况：质量完美、无违规、速度快，约为 `0.7`。
- 最差可行情况：质量差、软约束 100% 违规、速度慢，但会被封顶到 `< 10`，即 `M - 10^-6`。
- 不可行解获得 `M = 10`，通常高于任何合理可行解，但不会极端到一次失败支配整个总分。

总分影响示例：

- 若参赛者在 1 个大测试用例（例如 120 个模块）上失败，会因为大规模实例权重较高而受到显著惩罚。
- 若参赛者在 1 个小测试用例（例如 21 个模块）上失败，惩罚较轻。

指数违规惩罚 `exp(βV)` 会快速放大代价：

| `Violationsrelative` | 代价乘数 |
| --- | --- |
| 0.0 | ×1.00 |
| 0.25 | ×1.65 |
| 0.5 | ×2.72 |
| 1.0 | ×7.39 |

运行时间因子被封顶奖励如下（负值表示更好）：

| `RuntimeFactor` | 效果 |
| --- | --- |
| `<= 0.31`，约快 3 倍或更多 | -30%，封顶 |
| `0.5`，快 2 倍 | -19% |
| `1.0`，中位数 | 基线 |
| `2.0`，慢 2 倍 | +23% |
| `4.0`，慢 4 倍 | +51% |

注意：运行过慢带来的惩罚不封顶，只有运行更快带来的收益会在 30% 处封顶。

示例得分：

| 场景 | 代价 |
| --- | --- |
| 完美解，快 10 倍 | 0.70 |
| 完美解，约快 3 倍 | 0.70 |
| 完美解，中位运行时间 | 1.00 |
| 10% 差距、25% 违规、快 10 倍 | 1.24 |
| 50% 软约束违规、中位运行时间 | 2.72 |
| 100% 软约束违规、慢 2 倍 | 9.09 |
| 不可行（任意硬约束违规） | 10.00 |

代价越低越好。硬约束违规会导致固定惩罚 `M = 10`，确保不可行解总是差于可行解，同时允许在 100 个测试用例上获得部分分数。

## 半周长线长（HPWL）

HPWL 定义为连接模块与端点之间、按连接权重加权的质心曼哈顿距离之和。

模块间 HPWL 使用两个相连模块质心之间的曼哈顿距离计算：

```text
HPWLint =
sum_i sum_{j>i} W(int)ij *
(
  |cxi - cxj| + |cyi - cyj|
)
```

其中 `cxi = xi + wi / 2`，`cyi = yi + hi / 2`，即模块 `bi` 的质心坐标。

外部连接使用模块中心到端点的曼哈顿距离计算：

```text
HPWLext =
sum_i sum_j W(ext)ij *
(
  |xi + wi/2 - xtj| + |yi + hi/2 - ytj|
)
```

其中 `(xtj, ytj)` 是端点 `tj` 的坐标。

基于差距的归一化：

```text
HPWLgap = ((HPWLint + HPWLext) - HPWLbaseline) / HPWLbaseline
```

值为 0 表示解的线长与基线完全一致；值为 0.15 表示解比基线差 15%。

## 包围盒面积

对于布局 `M = {(xi, yi, wi, hi) | i = 1, ..., k}`：

- 包围盒左下角：`xmin = min_i xi`，`ymin = min_i yi`。
- 包围盒右上角：`xmax = max_i(xi + wi)`，`ymax = max_i(yi + hi)`。
- `Areabbox = (xmax - xmin) × (ymax - ymin)`。

基于差距的归一化：

```text
Areagap_bbox = (Areabbox - Areabaseline_bbox) / Areabaseline_bbox
```

值为 0 表示解的面积与基线完全一致；值为 0.05 表示解比基线多使用 5% 面积。

## 违规代价

不同类型约束的违规计算方式不同：

- 单模块约束（边界）：每个模块要么满足（0），要么违规（1）。
- 分组约束（Grouping、MIB）：按组计数，依据碎片化程度（连通分量数减 1）或形状不一致程度（不同形状数减 1）计算。

```text
Violationsrelative =
(Vgrouping + Vboundary + Vmib) / Nsoft
```

其中 `Nsoft` 是归一化因子：

```text
Nsoft =
|Bboundary|
+ sum_p (|Gp| - 1)
+ sum_q (|Mq| - 1)
```

这里 `|Gp|` 是第 `p` 个 grouping 组的模块数，`|Mq|` 是第 `q` 个 MIB 组的模块数。该归一化确保 `Violationsrelative ∈ [0, 1]`。`v10` 明确将固定形状和预放置模块移出软约束统计，因为它们现在属于硬约束。

边界违规：若模块未接触要求的包围盒边或角，则计 1。

Grouping 违规：设 `P` 为组数，`Gp` 为第 `p` 个组的模块集合，`cp` 为 `Gp` 内模块形成的连通分量数（两个模块共享边即视为连通）。完全满足时 `cp = 1`。一个含 `|Gp|` 个模块的组最大违规为 `|Gp| - 1`，即所有模块彼此孤立：

```text
Vgrouping = sum_p (cp - 1)
```

多实例违规：设 `Q` 为 MIB 组数，`sq` 为第 `q` 个组内不同 `(w, h)` 尺寸对的数量。完全满足时 `sq = 1`。一个含 `|Mq|` 个模块的组最大违规为 `|Mq| - 1`，即所有模块形状都不同：

```text
Vmib = sum_q (sq - 1)
```

## 数据集

机器学习依靠可扩展 transformer 和大规模预训练数据集快速发展。然而，SoC 布局规划缺少可用于监督学习的类似数据集。我们提供 FloorSet-Lite 数据集（固定矩形外框内的矩形模块），其中布局是按构造方式保证最优的：

- 训练集：100 万个样本，包含从 21 到 120 个模块规模的最优解。参赛者可将其用于训练 ML 模型或分析问题结构。数据可在 Hugging Face 获取。可使用 `iccad2026_evaluate.py` 中的 `get_training_dataloader()` 自动下载并访问数据。
- 验证集：100 个样本，每种规模 1 个，从 21 到 120 个模块。参赛者可用于验证解的泛化能力。数据可在 Hugging Face 获取。可使用 `iccad2026_evaluate.py` 中的 `get_validation_dataloader()` 自动下载并访问数据。
- 测试集：100 个样本，每种规模 1 个，从 21 到 120 个模块，对候选者隐藏，用于最终提交评估。
- 仓库包含 PyTorch DataLoader、得分评估器、不可行性检查和绘图工具。强烈建议在提交前使用提供的评估器检查解。
- 数据集中为每个测试用例提供基线值 `HPWLbaseline` 和 `Areabaseline_bbox`。

## 总分（Cost）

总分是在 100 个隐藏测试样例上的加权平均代价。权重随实例规模指数增加：

```text
Total Score =
sum_{i=21}^{120} Cost[i] * exp(ni / 12) / sum_{j=21}^{120} exp(nj / 12)
```

其中：

- `Cost[i]` 是第 `i` 个测试用例按目标函数计算得到的代价。
- `ni` 是第 `i` 个测试用例的模块数；在该数据集中 `ni = i`。
- 等价地，每个测试用例 `i` 的代价乘以归一化权重 `λi = exp(ni / 12) / Z`，其中 `Z = sum_{j=21}^{120} exp(nj / 12)`，且所有 `λi` 之和为 1。

这种指数加权方案确保规模更大、难度更高的实例对最终得分贡献更大。

总分越低越好。若一个完美解在所有测试用例上达到基线指标并使用中位运行时间，总分会接近 `1.0`。

## 鼓励机器学习解决方案

本竞赛通过以下方式鼓励数据驱动的 ML 引导方案：

- 注重效率的评分：运行时间被显式纳入评分函数，并且按模块数指数加权会惩罚随问题规模增长而扩展性较差的方法。
- 可扩展性门槛：更大的实例对传统方法构成显著挑战，传统方法随着规模增大容易受到运行时间惩罚。能够从训练数据中学习的 ML 方法有潜力更高效地泛化到更大实例。

## 主要内容简述

这份文档是一个面向 ICCAD 2026 的 FloorSet/FloorSet-Lite SoC 布局规划竞赛说明。它把工业 SoC floorplanning 建模为带固定外框、多种软硬约束和多目标优化的组合优化问题，要求参赛者输出无重叠模块位置和尺寸，并尽量降低线长、包围盒面积、软约束违规和运行时间。

文档强调传统启发式方法在工业规模和复杂约束下收敛慢、难以达到最优，而机器学习引导方法可能通过大规模最优数据学习问题结构，从而快速生成高质量布局。竞赛提供 100 万训练样本、100 个验证样本和 100 个隐藏测试样本，并配套评估器、可行性检查和绘图工具。

评分函数的核心是：硬约束违规直接判不可行并给固定惩罚；可行解则按 HPWL 差距、包围盒面积差距、软约束违规比例和相对运行时间综合评分，并在 `M - 10^-6` 处封顶。`v10` 进一步把固定形状模块与预放置模块从软约束改为硬约束，把模块间 HPWL 改为按质心曼哈顿距离计算，并将总分权重改为 `exp(n / 12)`。因此算法不仅要解得好，还必须能扩展到 120 个模块规模并保持较快运行。
